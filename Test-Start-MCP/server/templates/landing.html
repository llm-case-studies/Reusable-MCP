Test‑Start‑MCP

Safely start and smoke‑test local MCP services from models when their sandboxes can’t run scripts. This service enforces allowlists, pre‑flight, overlays, and audits to keep human approval in control.

Quick Links (UIs)

  • MCP Playground — initialize, list tools, call tools
  • Runner UI — list allowed, run (REST), run (SSE), logs, stats/health
  • Admin — add/remove rules, assign overlays, view policy audit

Tips

  • Set TSM_TOKEN in localStorage to authenticate UIs.
  • Use check_script before run_script; when enforced, include preflight_token.
  • Assign session overlays in Admin to clamp runtime caps per project or path.
